# プロジェクト企画書


## 1. プロジェクト名
Minecraftサーバー構築・クラウド移行プロジェクト（仮）


## 2. 背景
（なぜこれを作るのか？インフラエンジニアを目指すきっかけや、マイクラを選んだ理由など）
インフラエンジニアで就職する際の強みとするため
幼少期に見て憧れ、ITのイメージを形作ったのがサーバーに囲まれて仕事する人だったから
友人にMaincraftサーバーの構築を頼まれたため


## 3. 目的
（このプロジェクトを通して、自分は何を達成したいのか？）
Maincraftのサーバー運営を通して、仮想とクラウドの双方でサーバー構築・管理の経験得る。


## 4. 対象者
（このマイクラサーバーは誰が利用するものか？友達、自分、など）
自身と友人　　合わせて5人程度

## 5. ゴール
（完成したら、システムや自分自身がどんな状態になっていれば成功か？）
AWS上にサーバーを作成
　稼働させた日の停止時と日付が変わるころに自動でバックアップを取る
　障害が発生した際にバックアップを元に復旧できる
　自身がいないタイミングでの障害に気づけるようにディスコードで監視する。
　バックアップを古い順に削除する　（保存期間と件数は未定）
