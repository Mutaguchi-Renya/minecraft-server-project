# プロジェクト企画書

## 1. プロジェクト名
Minecraftサーバー構築・クラウド移行プロジェクト（仮）

## 2. 背景
* Minecraftサーバーは継続的な運用管理が必要となる一方、小規模環境における運用自動化の実践機会は限られている。本プロジェクトでは実サービスを想定したサーバー構築・運用を行い、インフラ技術の習得を目的とする。
* 就職活動において、自身の技術力と実践的なアプローチを証明できる強み（ポートフォリオ）を作成するため。
* 友人より実用的なゲームサーバーの構築を依頼されたことを機に、実際の業務案件に見立てたインフラ構築・運用のプロジェクトを発足した。

## 3. 目的
* Minecraftサーバーの構築・運用を通じて、Linuxサーバーの基礎、仮想化技術、およびクラウド環境の選定・移行・運用の実践的なスキルを習得する。
* 単なるサーバー構築に留まらず、バックアップの自動化や監視体制の構築といった「インフラの運用保守」の実務プロセスを総合的に経験・理解する。

## 4. 対象者
* 自身および友人（計5名程度）

## 5. スコープ
本プロジェクトにおける作業範囲およびシステムの境界線を以下の通り定義する。

### 対象（インフラ構築・運用保守の範囲）
* Linuxサーバー（OS環境）の構築およびセキュリティ設定
* Minecraftサーバーの導入とシステムサービス化（自動起動等の管理）
* シェルスクリプト等を用いた運用自動化（バックアップ・世代管理）
* サーバー監視および異常時の通知体制の構築
* オンプレミス（ローカル）環境からクラウド環境への移行プロセスの検証
* 一連の設計・運用ドキュメントの作成、およびGitHubによる構成管理

### 対象外（本プロジェクトでは実施しない範囲）
* 不特定多数への完全な一般公開（認証情報を持つ特定メンバーのみを対象とする）
* 営利目的の商用運用、および課金システムの導入
* 大規模アクセスを想定した負荷テストや冗長化構成の構築
* ゲーム内（Minecraft内）のプラグイン開発やModの独自カスタマイズ

## 6. ゴール
1. **クラウド移行の完了**: クラウド環境へ移行し、正常に動作すること。
2. **運用の自動化**: サーバー停止時および日次（日付変更時）の自動バックアップ体制を確立すること。
3. **可用性と復旧性の確保**: 障害発生時にバックアップから迅速に復旧できる手順を確立し、実証すること。
4. **監視体制の構築**: サーバー監視および通知体制を構築する。
5. **ライフサイクル管理**: ストレージ容量を圧迫しないよう、バックアップファイルの世代管理（古いファイルの自動削除）を実装すること。
