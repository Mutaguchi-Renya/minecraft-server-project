# プロジェクト企画書

## 1. プロジェクト名
Minecraftサーバー構築・クラウド移行プロジェクト（仮）

## 2. 背景
* 幼少期に目にした「サーバーに囲まれて働くエンジニア」への憧れが自身のITへの原風景であり、インフラエンジニアを志望する強い動機となっている。
* 就職活動において、自身の技術力と実践的なアプローチを証明できる強み（ポートフォリオ）を作成するため。
* 友人より実用的なゲームサーバーの構築を依頼されたことを機に、実際の業務案件に見立てたインフラ構築・運用のプロジェクトを発足した。

## 3. 目的
* Minecraftサーバーの構築・運用を通じて、Linuxサーバーの基礎、仮想化技術、およびクラウド（AWS）環境の選定・移行・運用の実践的なスキルを習得する。
* 単なるサーバー構築に留まらず、バックアップの自動化や監視体制の構築といった「インフラの運用保守」の実務プロセスを総合的に経験・理解する。

## 4. 対象者
* 自身および友人（計5名程度）

## 5. ゴール
1. **クラウド移行の完了**: AWS上に安定して稼働するMinecraftサーバー環境を構築すること。
2. **運用の自動化**: サーバー停止時および日次（日付変更時）の自動バックアップ体制を確立すること。
3. **可用性と復旧性の確保**: 障害発生時にバックアップから迅速に復旧できる手順を確立し、実証すること。
4. **監視体制の構築**: Discordと連携し、管理者が不在時でもサーバーの異常（ダウン検知等）を即座に通知・把握できる仕組みを作ること。
5. **ライフサイクル管理**: ストレージ容量を圧迫しないよう、バックアップファイルの世代管理（古いファイルの自動削除）を実装すること。
